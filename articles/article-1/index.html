Basic Data Science Interview Questions

July 27, 2021

Harvard Business Review referred to data scientist as the “sexiest job of the 21st century.” Glassdoor ranked it the #2 job in America for 2021. It’s no wonder that in the new era of big data and machine learning, data scientists are becoming rock stars.

For this very reason, the data science field is highly competitive, and getting a first interview for those of us with little or no prior experience in the field can be very challenging.

Selection processes usually have several stages, and it is very likely that in at least one of them we will have to answer a series of basic questions, solve a series of exercises, or explain how we would approach a certain situation.

In this post I want to provide some of the most common questions that may be asked in an interview for a junior data scientist. This will serve both to rehearse an answer beforehand, and to find out how polished our basic knowledge of data science and machine learning is.

This is not intended to be an exhaustive list of questions, as the possibilities are endless. However, if you want more information not only about the most important questions, but about the entire interview process for a Data Science position, I recommend reading this Springboard document, which you can find in my Github repository.

1. What is Logistic Regression? What is the difference with Linear Regression?

Logistic Regression is a Machine Learning algorithm which used for classification where we have 2 possible outcomes (1 or 0). It’s a predictive analysis algorithm and based on the concept of probability.

Logistic Regression is a special case of linear regression where the target variable is categorical in nature. Linear regression gives you a continuous output (like predicting house price and stock price), but logistic regression provides a constant output (like predicting whether a patient has cancer or not).

Other concepts that you should understand on this point are:

  • Sigmoid function
  • Binary vs. multinomial vs. ordinal logistic regression
  • Decision boundary
  • Cost function

2. What is a Confusion Matrix?

A confusion matrix is a table that is often used to describe the performance of a classification model (or “classifier”) on a set of test data where output can be two or more classes. It is a table with 4 different combinations of predicted and actual values.

For example, if we are evaluating a model that aims to predict whether a person has cancer or not, the results of the confusion matrix would be interpreted as follows:

  • True positives (TP): These are cases in which we predicted yes (they have cancer), and they do have the disease.
  • True negatives (TN): We predicted no, and they don’t have the disease.
  • False positives (FP): We predicted yes, but they don’t actually have the disease. (Also known as a “Type I error.")
  • False negatives (FN): We predicted no, but they actually do have the disease. (Also known as a “Type II error.")

From this data we can calculate different metrics:

  • Recall: TP/(TP+FN) from all the positive cases, how many we predicted correctly.
  • Precision: TP/(TP+FP) from all the cases we have predicted as positive, how many are actually positive.
  • Accuracy: (TP+TN)/(TP+TN+FP+FN) from all the cases (positive and negative), how many of them we have predicted correctly.

4. What are the differences between supervised and unsupervised learning?

To put it simply, the main distinction is that supervised learning uses labeled input and output data (this means some data is already tagged with the correct answer), while an unsupervised learning algorithm does not.

In supervised learning, the algorithm “learns” from the training dataset by iteratively making predictions on the data and adjusting for the correct answer. They require upfront human intervention to label the data appropriately (also this can be done by a previous unsupervised learning algorithm). They are ideal for applications such as spam detection, sentiment analysis, weather forecasting and pricing predictions, among other things. Some examples of supervised learning models are decision trees, linear or logistic regression, and support vector machine.

In contrast, unsupervised learning models work on their own to discover the inherent structure of unlabeled data (however, they still require some grade of human intervention). This models are used for anomaly detection, recommendation engines, customer clustering and medical imaging. K-means clustering and hierarchical clustering are come examples of this type of models.

5. What is overfitting and how can you avoid it?

Overfitting occurs when you achieve a good fit of your model on the training data, while it does not generalize well on new, unseen data. In other words, the model learned patterns specific to the training data, which are irrelevant in other data.

We can identify overfitting by looking at validation metrics like loss or accuracy. Usually, the validation metric stops improving after a certain number of iterations and begins to get worse. The training metric continues to improve because the model seeks to find the best fit for the training data.

Some techniques to prevent overfitting are:

  • Get a simpler model: the simpler the model is, the less likely it will be able to overfit the training set.
  • Early stopping: this is a form of regularization while training a model with an iterative method, such as gradient descent. This method update the model so as to make it better fit the training data with each iteration. After some time, improving the model’s fit to the training data leads to increased generalization error. Early stopping rules provide guidance as to how many iterations can be run before the model begins to overfit.
  • Get more training data: unfortunately, in real-world situations, you often do not have this possibility due to time, budget, or technical constraints. In some applications, such as image recognition, you can artificially augment the training data by using Data Augmentation (applying flipping, translation, rotation, scaling, changing brightness, adding noise, etc. to the original image).
  • Use Regularization: this can help to get a simpler model too. It does so by adding a penalty term to the loss function.

6. How can we deal with Outliers?

An outlier is a data point that is distant from other similar points. The most common causes of outliers on a data set are:

  • Data entry errors (human errors)
  • Measurement errors (instrument errors)
  • Experimental errors (data extraction or experiment planning/executing errors)
  • Intentional (dummy outliers made to test detection methods)
  • Data processing errors (data manipulation or data set unintended mutations)
  • Sampling errors (extracting or mixing data from wrong or various sources)
  • Natural (not an error, novelties in data)

Here are some of the most popular methods for outlier detection. I recomend taking a time to understand each technique and it’s possible applications:

  • Z-Score or Extreme Value Analysis (parametric)
  • Probabilistic and Statistical Modeling (parametric)
  • Linear Regression Models (PCA, LMS)
  • Proximity Based Models (non-parametric)
  • Information Theory Models
  • High Dimensional Outlier Detection Methods (high dimensional sparse data)
